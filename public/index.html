Guide

Description Of Usual Server Responses:

  • 200 OK - the request was successful (some API calls may return 201 instead).
  • 201 Created - the request was successful and a resource was created.
  • 204 No Content - the request was successful but there is no representation to return (i.e. the response is empty).
  • 400 Bad Request - the request could not be understood or was missing required parameters.
  • 401 Unauthorized - authentication failed or user doesn't have permissions for requested operation.
  • 403 Forbidden - access denied.
  • 404 Not Found - resource was not found.
  • 405 Method Not Allowed - requested method is not supported for resource.

Allowed HTTPs requests:

  • GET - Get a resource or list of resources.
  • POST - To create resource.
  • PUT - Update resource.
  • DELETE - To delete resource.

Router.post

Sign up

URL -> /users/

Sign up Created
Status 201
 Sign up Failed
Status 400
  • User - min 2 chars, max 30 chars.
  • Email - min 2 chars, max 50 chars.
  • Password - min 2 chars, max 100 chars, hidden.
  • Role - admin/reguler -> default = regular
  • Date Created - default = Date.now

Router.post

Login

URL -> /users/login

Login Succeed
Status 200
 Login Failed
Status 400
  • Email - valid email
    else -> "Email Incorrect!"
  • Password - valid password
    else -> "Password Incorrect!"

    • After login the user getting Token for 60 minutes!

Products Routes

Click on the routes for more information

Router.get

URL -> /products

List of 10 products in shop
  • url/products?page=0
    - list of 10 first products
  • url/products?page=1
    - list of 10 (or the rest) of the next products
  • url/products?sort=name
    - sorting the list by the name, can be replaced to any other parameters such as price, date etc.
  • url/products?=sort=name&reverse=yes
    - showing the list from the last to the first by the value of the sort.

Router.get

URL -> /products/search

List of 10 products in shop
  • url/products/search?s=NAME
    - Searching product by name.
  • url/products/search?s=INFO
    - showing one or more by the name or information of the product you insert.

    • Page,reverse and sort works here also if needed.

Router.get

URL -> /products/prices

List of 10 products in shop sorting by the price
  • url/products/prices?min=Number&max=Number
    - list of 10 products from the minimum price to the maximum price the user entered
  • url/products/prices?min=Number&max=Number&sort=price
    - sorted from the first to the last by price
  • url/products/prices?min=Number&max=Number&sort=price&reverse=yes
    - sorted from the last to the first by price

    • Page,reverse and sort works here also if needed.

Router.get

URL -> /products/category/

  • url/products/category/CATEGORY-NAME
    - list of 10 products showing by the category name

    • Page,reverse and sort works here also if needed.

Router.post

Add New Product

URL -> /products/

New product Created
Status 201
 New product Failed
Status 400

Correct table for adding new product-

  • Name - min 2 chars, max 50 chars, required
  • Info - min 2 chars, max 200 chars, required
  • Category - min 2 chars, max 50 chars, required
  • Image - min 2 chars, max 200 chars, required
  • Price - min 0 char, max 100 chars, required
  • Date Created - automatically will be added or admin can add it himself.

User can Add product when logged in with Token ONLY!
